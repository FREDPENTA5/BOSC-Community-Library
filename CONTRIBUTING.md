# Contributing to BOSC Community Library

Thank you for your interest in contributing! This project follows a professional development lifecycle.

## Development Workflow
1. **Issue Discussion**: All changes must start with an Issue.
2. **Branching**: Create a feature branch from `main`.
   `git checkout -b feature/your-feature-name`
3. **Draft PR**: Open a Pull Request early for feedback.
4. **Peer Review**: At least one maintainer review is required before merging.

## Technical Standards
- **Python**: 3.9+ required.
- **Type Hinting**: All new functions must include type hints.
- **Tests**: `pytest` is used for all unit testing.

## Commit Message Format
We follow the Conventional Commits specification:
- `feat:` for new features
- `fix:` for bug fixes
- `refactor:` for code changes that neither fix a bug nor add a feature
